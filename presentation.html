Learnings

What I discovered along the way

⚡

CLI + Browser = Superpowers

Give agents access to the command line and a browser and they become incredibly capable. The combination lets them research, build, test, and verify — all autonomously.

💬

From App to Chat

Chatting is just easier. Instead of navigating a complex UI, you describe what you want and the agent figures out the how. Natural language becomes the interface.

🔍

Let Claude Cook

Requesting things through Claude adds everything Claude can do to your Squarespace site. It searches Gmail for requests, executes edits, and handles attachments.

🔧

From Browser to CLI

Agents reverse-engineered an internal API so that Claude could skip the browser entirely. Slow, fragile automation became fast, reliable API calls.

🎤

Remote Control & Voice

Update Squarespace by talking to Claude on your phone. Voice input turns a commute into productive client work.

📦

Offload Everything to Claude Desktop

Don't build features — build tools and let Claude Desktop compose them. The more you offload, the more capable the system becomes.

🤖

Dynamic Agents, Not Static Code

Instead of hardcoding logic, let agents decide how to accomplish tasks. They adapt, compose tools on the fly, and handle edge cases you'd never anticipate.

💰

Claude Code CLI = Free Agents

Run agents through Claude Code's CLI using your Max subscription instead of API tokens. Unlimited agents for a flat monthly fee.

🤓

Use Claude Code in Terminal

It's streets ahead. The terminal gives you power, speed, and flexibility that no GUI can match.

🔄

Workflows → MCPs, Approaches → Skills

Turn repeatable workflows into MCP tools so agents can execute them. Turn repeatable approaches into skills so agents know how to think.

The Key Distinction

Skills teach thinking, MCPs enable doing

Skill MCP
Encodes Knowledge & judgment Capability & action
Output Guides Claude's reasoning Executes something real
Lives in A markdown file A running server
Fails if Claude ignores it The server is down
Use a Skill when:

Claude keeps getting the approach wrong — you need to teach it the right pattern or workflow.

Use an MCP when:

Claude knows what to do but needs the tools to actually do it — API calls, file operations, external services.

Use Both together:

The ideal combo. A skill encodes the workflow logic (like the two-step blog post pattern), while the MCP gives Claude the tools to execute it.
